Description

Machine learning algorithms can handle numerical features, but when it comes to categorical features, they require transformation into numeric values before they can be utilized. The type of transformation depends on whether the categorical feature has a natural order (ordinal feature) or not (nominal feature).

Numerical features with different scales can have varying degrees of influence on a machine-learning model, which can skew the results. These features can be transformed to have the same scale to address this. This process can improve the performance of the model.

In this stage, you will transform the training, validation, and testing sets into a form that the machine learning algorithms can understand.

Objectives

To complete this stage:

  1. Assemble the numerical and categorical transformation steps in a ColumnTransformer. Use StandardScaler for the numerical features and OneHotEncoder for the categorical features;
  2. Fit the ColumnTransformer on the training set;
  3. Transform the training, validation, and testing sets with the ColumnTransformer;
  4. Print the dictionary containing the shapes of the transformed training, validation, and testing sets.
